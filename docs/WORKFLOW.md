# Workflow

## Core Loop

1. Capture source evidence in `raw/` (or references to external material).
2. Append one entry to `wiki/log.md` for substantive work.
3. Update affected canonical wiki pages.
4. Refresh `wiki/overview.md` links if page set changed.
5. Record unresolved system work in `IMPROVEMENTS.md`.

## What Counts As Substantive

- decisions that affect future behavior
- architecture or process changes
- meaningful progress with durable implications
- blockers that constrain next steps

Routine chatter usually does not need a log entry.

## Suggested Cadence

- per substantive chat or work block: append to `wiki/log.md`
- weekly: review log and refresh high-traffic canonical pages
- monthly: prune stale pages and consolidate overlap

## Update Contract

For each substantive update:

- chronology: add one concise log entry
- synthesis: update one or more canonical pages
- provenance: include source references

## Page Promotion Rule

Create a new canonical page only when:

- topic recurs
- update cannot fit cleanly into an existing page
- page will likely be reused across future sessions

Otherwise, keep it in log-only form.
